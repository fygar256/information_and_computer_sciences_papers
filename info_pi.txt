Here, we assume that π is a perfectly random number sequence.

This program generates pi, decodes it into characters, and extracts the information contained in pi.

This program's decoding method uses the string "i=" 0123456789.,abcdefghijklmnopqrstuvwxyz" and divides it into two-digit decimal chunks, dividing each chunk by 100 and multiplying by the length of i, then assigning the result to i.

The information that can be extracted varies depending on the decoding method. Since π is a high-quality random number sequence, most of the information is useless and not practical.

This program generates π using a Spigot-type algorithm.

``info.py`
#!/usr/bin/env python3
# Generates π using a Spigot-type algorithm and decodes it into characters in two-digit chunks.
# Modification: Changed digit comparison and extraction from floating-point division (/) to integer floor division (//).

# The original a/b and a1/b1 approximate values ​​of π from above and below, so in practice they go quite deep.
# Although they produce the correct number of digits, (1) distinct rational numbers can be == in float,
# (2) the floor of int(a/b) shifts just below the integer boundary, which is fundamentally unhealthy.
# // Using this makes the correctness unconditionally strict, independent of IEEE-754 rounding or the digit statistics of π.
import sys
k, a, b, a1, b1 = 2, 4, 1, 12, 4
i = " 0123456789.,abcdefghijklmnopqrstuvwxyz"
buf = [] # 2-digit buffer. Maintain the value across the outer loop
while True:
# Next approximation
p, q, k = k*k, 2*k+1, k+1
a, b, a1, b1 = a1, b1, p*a + ​​q*a1, p*b + q*b1
# Decode common digits in disjoint 2-digit chunks
d = a // b
d1 = a1 // b1
while d == d1:
buf.append(d)
if len(buf) == 2:
# Convert (buf0*10+buf1)/100*len(i) to integer arithmetic (matches the float version in the range of 0..99)
n = (buf[0] * 10 + buf[1]) * len(i) // 100
print(i[n], end='')
sys.stdout.flush()
buf = []
a, a1 = 10 * (a % b), 10 * (a1 % b1)
d, d1 = a // b, a1 // b1

```

In this program's Spigot-type algorithm, the value of pi can be retrieved if enough time is available, allowing for the extraction of an infinite number of digits of pi from finite memory. However, the time given to humanity is finite. It would be interesting to assume it's infinite.

The amount of information contained in an event occurring with probability p is -log_2 p.
Therefore, the amount of information contained in the answer to a safe key quiz with a probability of 1/1000 is -log_2 1/1000 = 3log_2 10 = 9.96578…

If an event occurring with probability 0 occurs, the amount of information is infinite, and if an event occurring with probability 1 occurs, the amount of information is 0.

The definition of "information amount" is:
① "Information amount" quantitatively defined in information theory.
② "Information amount" simply representing how many digits of information are arranged in one bit.

These two can sometimes be treated as one without contradiction.
The unit is [bit].

### Differences in information amount due to representation (encoding)

For example, if π is represented as "the ratio of the circumference to the diameter of a circle," it is 27 bytes in UTF-8 code. However, if it is 3.141592…, it has infinite information. Simply writing "π" is 2 bytes in UTF-8 code. Writing "pi" is 2 bytes in ASCII code. These refer to the same thing, although there is a distinction between content and container. If π = 3.14, then "3.14" is 4 bytes in ASCII code.
As a 64-bit floating-point number, 3.14 ≈ "0x1f 0x85 0xeb 0x51 0xb8 0x1e 0x09 0x40" is 8 bytes. This shows that different encoding rules result in different amounts of information.
